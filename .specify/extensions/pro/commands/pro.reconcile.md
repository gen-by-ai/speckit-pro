# SpecKit Pro — Spec Drift Reconciliation

Structured **spec-vs-reality** review after `/speckit.implement`. Addresses the core gap between static Spec Kit artifacts and platform-style **living specs**: `spec.md`, `plan.md`, and `tasks.md` do not update themselves when code diverges.

This command **does not** silently rewrite specs. It produces a factual drift record so humans or follow-up `/speckit.clarify` / `/speckit.specify` runs can align documents deliberately.

**Recommended position:** run immediately **after** implementation (and optional commit) and **before** `/speckit.pro.evaluate`, so the evaluator can weigh known drift.

## User Input

```text
$ARGUMENTS
```

Optional flags (parse if present):

| Flag | Meaning |
|---|---|
| `--refresh-repo-ai` | If `repo-ai` CLI is available, run `repo-ai build` once before retrieval queries (slow; refreshes local embeddings) |

## Prerequisites

1. Run `.specify/scripts/bash/check-prerequisites.sh --json` from the repo root and parse **`FEATURE_DIR`** (absolute path to the active feature spec directory).

2. Verify **`FEATURE_DIR`** contains at least `spec.md` (and ideally `plan.md`, `tasks.md`).

## Retrieval (optional but recommended)

If the repository includes **`repo-ai`** (`repo-ai/package.json` or global `repo-ai` on `PATH`) and the index exists (**`repo-ai/vectordb/index.json`**):

- Run **`repo-ai search`** with 2–4 short queries derived from the feature title and top user stories (see `.agents/skills/repo-ai-cli/SKILL.md`).
- Use hits only as **navigation hints** — still open primary sources (`spec.md`, source files).

If no index exists and `--refresh-repo-ai` was **not** passed, skip retrieval (do not block).

## Drift Analysis

### 1. Implementation surface

From git (repo root):

```bash
git diff --name-only HEAD~1   # or wider range if multi-commit sprint
git status --short
```

Relate changed paths to **`plan.md`** components and **`tasks.md`** checkboxes.

### 2. Compare documents to code

For each **P1 user story** (and key P2 items) in **`spec.md`**:

- **ALIGNED** — behavior clearly implemented; paths or tests cited.
- **DRIFT** — implementation differs from the spec text (different behavior, extra constraints, missing acceptance criterion).
- **UNKNOWN** — cannot verify without runtime test or missing files.

Repeat for **`plan.md`** architectural bullets if they are falsifiable (stack choices, module boundaries).

Cross-check **`tasks.md`** completed items: ensure done tasks are actually reflected in the tree (no false completions).

### 3. Sprint contract

If **`<FEATURE_DIR>/contracts/`** or **`.ai-knowledge/<feature>/contracts/`** exists, note whether any **acceptance criterion** may be obsolete given DRIFT items (do not delete contract files here — flag only).

## Output

Write **`FEATURE_DIR/pro-drift.md`** (overwrite each run):

```markdown
# Spec drift — <feature-name>

> Generated by SpecKit Pro Reconcile | <ISO timestamp>
> Feature dir: <FEATURE_DIR>

## Summary

- **Aligned:** <n>
- **Drift:** <n>
- **Unknown:** <n>

## Findings

| Story / area | Status | Evidence |
|--------------|--------|----------|
| … | ALIGNED \| DRIFT \| UNKNOWN | File paths, tests, or notes |

## Recommended follow-ups

- [ ] **If DRIFT > 0:** Run `/speckit.clarify` or targeted `/speckit.specify` update; or edit `spec.md` / `plan.md` manually.
- [ ] **Before merge:** Ensure `/speckit.pro.evaluate` considers drift rows marked DRIFT.

## Retrieval notes (optional)

Brief bullets if `repo-ai search` was used (queries + top paths).

```

## Output Protocol

End stdout with:

```
[Pro] Drift reconciliation complete — wrote FEATURE_DIR/pro-drift.md (A aligned, D drift, U unknown).
```

If **`FEATURE_DIR`** cannot be resolved:

```
[Pro] ERROR: Could not resolve FEATURE_DIR — run from a Spec Kit feature workspace or pass prerequisites.
```

## Hook Behavior

When fired from **`after_implement`**, this command runs **before** `speckit.pro.evaluate` so evaluation can reference **`pro-drift.md`** if present.
